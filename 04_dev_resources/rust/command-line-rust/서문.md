- 커맨드 라인을 rust로 작성한다.
- rust의 공식문서를 읽는것에 대해 회의감이 있었기 때문에 러스트로 커맨드라인을 작성해보는 것으로 돌파하려 한다. 
- rust는 누구나 안정적이고 효율적인 소프트웨어를 만들수 있도록 힘을 실어주는 언어다.
- rust는 오브젝트 지향언어는 아니지만 다양한 타입을 내포할수 있는 struct와 타입의 행동을 정의한 trait이 있다.
	- 상속은 할수 없지만 struct(구조체)는 다른 struct를 인자로 받아서 사용할 수 있다.
	- trait가 다른 trait를 사용하고자 한다면 "이 행동을 하려면 저 행동도 할 줄 알아야 한다"는 식으로 의존 관계(Bound)를 정의한다.
```rust

// 1. 걷는 행동 정의
trait Walk {
	fn walk(&self);
}

// 2. 뛰는 행동 정의 (뛰려면 걸을 줄도 알아야 함)
trait Run: Walk {
	fn run(%self);
}

// Run trait가 Walk를 상속받는 것처럼 보이지만, 데이터(프로퍼티)를 물려받는 게 아니라
// "내가 Run을 구현하고 싶다면, Walk 기능도 반드시 내 손으로 구현해 놔야 한다"는 
// 규칙만 강제하는 것이다.
```

- cargo는 package manager,  build tool, test runner이다.
- `cargo new <dir명>` 으로 rust 프로젝트를 생성할 수 있다.
- file은 어떤 종류의 파일포맷인지 판별해주는 유닉스/macos/리눅스 표준 터미널 명령어다.
- tree는 현재 위치한 디렉토리를 기준으로 해서 그 하위에 있는 모든 디렉토리와 파일들을 계층적인 트리 구조의 텍스트로 시각화해주는 명령어다.
- `cli.rs` 라는 파일을 만들고 그 파일 안에 `#[test]`라고 속성을 부여해주면, `cargo test`를 실행할 때 그 파일을 테스트파일이라고 인식하고 테스트를 실행한다.
	- (추가로 배운건데 `tree -L 2` 이라고 입력하면, 현재 위치한 디렉토리에서 깊이가 2인만큼 하위의 디렉토리와 파일을 시각화해준다.)
- OS는 커맨드가 실행되면 그 커맨드의 이름으로 `$PATH`(미리 정의된 디렉토리 집합)에서 해당하는 디렉토리를 찾는다.
- `enum`은 서로 상관되어 있는 상수들의 집합을 정의할 수 있는 특별한 데이터 타입이다.
	- 이 변수가 가질 수 있는 값은  오직 이 몇가지 중 하나다, 라고 정의한거다.
	- 0, 1, 2 라는 숫자가 있을 때 0이 east, 1이 west라고 약속을 하고 사용하면, 나중에 봤을 때 0이 무슨 의미인지 알 수가 없기 때문에 정의해놓는다.
	- 정의된 값 이외에는 절대 대입할수 없다.
- `variant`는 열거형이 가질수 있는 여러 상태나 값이다.
	- 다른 언어와는 다르게 rust에서는 숫자에 이름을 붙이는 게 아니라 직접 데이터를 갖는다.
```rust
enum IpAddKind {
	V4(String),
	V6(String),
}

let home = IpAddKind::V4::(String::from("127.0.0.1"));
```
- `tr`을 사용하면 문자를 대체할 수 있다.
	- ex
		- `echo $PATH | tr : '\n'`
- 개발하면 iterm -> lazyvim을 사용하고 있는데 iterm -> lazyvim -> tmux를 사용하는게 아니라, Iterm -> tmux -> ctrl + b, %로 화면 분할을 한뒤, ctrl + b, h로 오른쪽 화면으로 이동 -> nvim으로 lazyvim을 실행 -> 해당 파일을 열기를 하면 나에게 이상적인 환경을 사용할수 있다.
- cargo test를 실행하면 test 파일에 정의된 `std::process:Command`를 사용했을 때에는 환경변수에 등록되어있는 디렉토리들에서 command를 찾느라 현재 crate 디렉토리의, `target/debug/xxx`를 찾을수 가 없는데 `assert_cmd`를 사용하면 해결할 수 있다.
