- 프로그램이 실행되는 도중에(runtime) 발생하는 에러
- 코드를 작성할 떄 문법이 틀린 것(컴파일 에러)과는 달리, 문법은 완벽하게 맞지만 실행 과정에서 논리적으로 불가능한 상황이 벌어졌을 때 프로그램이 멈추는 것
- 비유: 요리 레시피
	- 컴파일 에러(문법 에러)
		- 레시피의 글자가 깨져서 "소금 1스푼"인지 "설탕 1스푼"인지 읽을 수 없는 상태
			- 컴퓨터가
		- 이 경우, 요리 자체를 시작 할 수 없다.
		- (예: `if (a`처럼 괄호를 닫지 않는 실수)
	- 런타임 에러(실행 중 에러)
		- 레시피는 "달걀 3개를 깨서 넣으세요"라고 완벽하게 쓰여있다.
		- 하지만 요리를 하려고 냉장고를 열었더니 달걀이 하나도 없었다.
		- 레시피 자체는 정상이지만, 실제 실행(요리) 중에 문제가 발생하여 요리를 계속할 수 없게 되었다.
- 예시
```javascript
// response가 undefined일 경우, TypeError가 발생.
if (response.status) {
	console.log(response.status)
}

```
