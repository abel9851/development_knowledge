### 오늘 배운 것
- axios.interceptors를 실행하는 도중 cannot read properties of undefined (reading '\_retryCount\') 라는 에러가 발생했다.
	- `axios.interceptors.request.use`에서 config를 return하지 않으면 undefined를 반환한다는 것을 알았다.
	- axios의 내부에서 config.cancelToken을 사용하려하고 이때 cannot read propeties of undefiend (reading 'cannotToken')이 throw되고 `axios.interceptors.response.use`의 error객체에 TypeError가 할당된다는 것을 알았다.
	- javascript코드에서 이게 왜 발생하는지를 정확히 알기 힘들기 때문에 처음부터 config를 반환해야한다는 `axios.inteceptors.request.use`의 사양을 공식문서를 통해 먼저 알고 있거나, chrome 개발자 도구의 디버거를 사용해서 빠르게 검증하고 이유를 유추하고 실험해봐야한다는 것을 알았다.
		- 이때 javascript에서 return 키워드를 사용해서 값을 return하지 않는다면 undefined가 리턴되는 것, undefined.{property}로 접근하면 `Uncaught TypeError: cannot read properties of undefined (reading '{property}')`가 발생한다는 것
		- `const a = {value:1}` 에 대해 `a.vlaue_2 = 2`로 해도 수정이 가능하다는 것.(const는 변수 자체를 상수로 만드는 것이 아니라 변수가 가리키는 참조(주소)를 상수로 만드는 것, 즉 변수 f는 실질적인 값이 아닌, 주소를 저장하는 것이다. 재할당만 막는다.)
		- `1.rest`와 같이 object에 존재하지 않는 property에 접근하면 undefIned가 출력된다는 것
		- `1.rest.rest`를 하면 1.rest는 undefined이니 undefined.rest가 되어 Uncaught TypeError가 발생한다는 것
		- Promise에서 throw된 error는 error변수에 할당된다는 것
		- Uncaught, Caught error는 chrome개발자 도구 디버거를 실행해서 바로 잡을 수 있다는 것
		- `const a = 1;`에서 `typeof a.rest.rest`를 하면 typeof연산을 하기 전에(typeof연산은 값이 특정된 후에 실행된다.) a.rest.rest의 error가 throw되서 typeof 연산이 실행되지 않는다는 것
		- `a.rest instanceof undefined`를 하면 `Uncaught TypeError: Right-hand side of 'instanceof' is not an object`가 발생한다는 것.
			- 이는 undefined가 Instance가 아니라 primitive value(원시값)이라는 것
			- 이때 사용하는 것은 엄격한 비교`(===)`를 사용해야한다는 것
				- `a.rest === undefined` 혹은 `typeof smoevalue === "undefined"`
			
### 오늘 해결한 문제
- axios.interceptors를 학습 중에 발생한 `TypeError: cannot read properties of undefined (reading '_retryCount')` 의 원인이 `axios.interceptors.request.use`에서 config를 return하지 않았기 때문이라는 것을 알고 해결했다.
