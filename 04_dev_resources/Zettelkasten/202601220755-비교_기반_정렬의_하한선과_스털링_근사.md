## [ID: 202601220755] 비교 기반 정렬의 하한선과 스털링 근사

**태그:** #algorithm #sorting #complexity #stirlings-approximation #information-theory

### 본문

비교 기반 정렬 알고리즘이 아무리 효율적이어도 $O(n \log n)$보다 빠를 수 없는 이유를 결정 트리 모델과 스털링 근사로 설명함.

1. **결정 트리 모델 (Decision Tree Model):**
    
    - 모든 비교 정렬은 `if a < b` 식의 이진 선택(Binary Choice)으로 정답을 찾아감.
        
    - $n$개의 원소를 정렬하는 모든 경우의 수(순열)는 $n!$개임.
        
    - 모든 정답에 도달하려면 트리의 리프 노드 수가 $n!$개 이상이어야 하므로, 높이 $h$에 대해 $2^h \ge n!$ 성립.
        
    - 즉, 질문 횟수(높이) $h$는 최소 $\log_2(n!)$ 이상이어야 함.
        
2. **스털링 근사 (Stirling's Approximation) - 직관적 이해:**
    
    - $\log_2(n!)$을 다루기 쉬운 Big-O 단위로 번역하는 도구.
        
    - **직관:** $n!$은 $n$ 근처의 숫자들을 $n$번 곱한 것($n^n$)과 유사한 증가율을 가짐.
        
    - **수학적 근사:** $\log(n!) \approx n \log n - n \log e$.
        
    - 상수를 제외하면 차수는 $O(n \log n)$이 됨.
        
3. **결론 ($\Omega(n \log n)$):**
    
    - 비교를 통해 정보를 얻는 모든 정렬은 $n \log n$ 비트만큼의 정보를 추출해야 함.
        
    - 따라서 $O(n \log n)$은 알고리즘의 최선을 넘어선, 문제 자체의 **물리적 하한선(Lower Bound)**임.
