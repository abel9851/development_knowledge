## [ID: 202601242215] 대칭 인덱스 계산과 탐색 비용 절감

**태그:** #algorithm #math #index-calculation #optimization #time-complexity

### 본문

배열 뒤집기(Reverse)와 같은 대칭적 작업에서 투 포인터가 효율적인 이유는 **인덱스 간의 수학적 관계**를 활용하여 탐색 비용을 $O(1)$로 줄이기 때문임.
거꾸로 말하자면 배열 뒤집기와 같은 대칭구조를 사용한다면 양 끝에 있는 인덱스를 수학적으로 계산할 수 있어서 brute force로 모든 경우를 찾을 필요가 없이 two pointer를 사용해서 한번 순회에 2개씩 처리할 수 있음.

1. **대칭 인덱스 공식:**
    
    - 배열 길이 $n$에서 인덱스 $i$의 대칭점 $j$는 항상 $j = n - 1 - i$를 만족함.
        
    - $i + j = n - 1$이라는 상수를 유지함.
        
2. **탐색(Search) vs 계산(Calculation):**
    
    - **탐색:** 조건에 맞는 요소를 찾기 위해 배열을 순회함 ($O(n)$).
        
    - **계산:** 수식을 통해 위치를 즉시 특정함 ($O(1)$).
        
3. **알고리즘적 이득:**
    
    - 바꿀 대상을 계산으로 즉시 찾아낼 수 있으므로, 중첩 루프 없이 단일 루프($O(n)$)만으로 전체 배열의 순서를 뒤집을 수 있음.
