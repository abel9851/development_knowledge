## [ID: 202601242145] In-place 알고리즘: 메모리 효율적 데이터 처리

**태그:** #algorithm #computer-science #memory-management #optimization #space-complexity

### 본문

**In-place 알고리즘**은 입력 데이터 구조를 직접 수정하여 결과물을 만들어내는 방식으로, 입력 크기에 비례하는 추가적인 메모리 공간을 사용하지 않는 알고리즘을 뜻함.

1. **공간 복잡도 (Space Complexity):**
    
    - 추가적인 메모리 할당이 입력값 $n$에 관계없이 일정함 ($O(1)$).
        
    - 단, 포인터나 인덱스 변수, 스왑을 위한 임시 변수 등 상수 개의 변수 사용은 허용됨.
        
2. **핵심 특징:**
    
    - **원본 수정:** 외부에서 전달된 원본 데이터(예: 리스트)의 내용을 직접 변경함 (Side Effect 발생).
        
    - **메모리 효율:** 데이터가 매우 클 때 새로운 복사본을 만드는 비용(Memory Allocation)을 절약하여 Out of Memory 에러를 방지함.
        
3. **비교:**
    

|**구분**|**In-place**|**Out-of-place**|
|---|---|---|
|**추가 메모리**|$O(1)$|$O(n)$|
|**데이터 보존**|원본이 파괴됨|원본이 유지됨|
|**대표 예시**|Quick Sort, Selection Sort|Merge Sort, 리스트 슬라이싱|

4. **주의사항:** 파이썬의 `str`처럼 **Immutable(불변)** 객체는 In-place 수정이 불가능함. 따라서 `List[str]`와 같은 **Mutable(가변)** 구조에서만 구현 가능함.
