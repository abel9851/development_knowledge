## 시간 복잡도 서열 (Fast to Slow)

|**복잡도**|**명칭 (Name)**|**설명 및 특징**|**대표 예시**|
|---|---|---|---|
|**$O(1)$**|**Constant**|데이터 양과 상관없이 즉시 실행.|Hash Map Lookup, Array Index Access|
|**$O(\log n)$**|**Logarithmic**|실행 시마다 처리할 데이터가 절반으로 줄어듦.|Binary Search|
|**$O(n)$**|**Linear**|데이터 양에 비례하여 선형적으로 증가.|For loop (single), Linked List traversal|
|**$O(n \log n)$**|**Linearithmic**|효율적인 정렬 알고리즘의 표준.|Merge Sort, Quick Sort, Heap Sort|
|**$O(n^2)$**|**Quadratic**|데이터 양의 제곱만큼 증가. (중첩 루프)|Bubble Sort, Insertion Sort, 2D Array Traversal|
|**$O(n^3)$**|**Cubic**|3중 중첩 루프. 데이터가 조금만 커져도 위험.|Matrix Multiplication (naive)|
|**$O(2^n)$**|**Exponential**|데이터가 늘어날 때마다 실행 시간이 배로 증가.|Fibonacci (recursive), Power Set|
|**$O(n!)$**|**Factorial**|가장 느림. 모든 경우의 수를 다 따지는 경우.|Traveling Salesperson Problem (brute force)|

## 알고리즘 선택 가이드 (N 크기 기준)

실제 코딩 테스트나 시스템 설계 시 $N$의 크기에 따라 허용되는 최대 복잡도의 가이드라인입니다. (Python 기준 1초당 약 $10^7$ 연산 가정)

- **$N \le 10 \sim 15$**: $O(n!)$, $O(2^n)$ (모든 경우의 수 탐색)
    
- **$N \le 500$**: $O(n^3)$
    
- **$N \le 2,000 \sim 5,000$**: $O(n^2)$
    
- **$N \le 10^5 \sim 10^6$**: $O(n \log n)$ (정렬 연산의 한계치)
    
- **$N \le 10^7 \sim 10^8$**: $O(n)$ (단일 루프)
    
- **$N > 10^8$**: $O(\log n)$, $O(1)$ (수학적 계산 또는 이진 탐색)
